Overview & Strategy

  • Goal: Rank on Google via two tactics — blog posts at scale (informational keywords) + service pages (money keywords: service × city).
  • Click distribution: ~40% of clicks go to position 1, ~20% to position 2, then drops fast. Higher rank = more leads.
  • Four pillars: (1) winning keywords, (2) on-page SEO, (3) technical SEO, (4) off-page SEO.
  • on-page SEO = content, structure, meta tags. technical SEO = site build, speed, crawlability, sitemap, robots.txt.
  • AI SEO = regular SEO. ChatGPT/Perplexity/Claude search Google first — rank on Google and you rank in AI answers.
  • Wrong strategy risk: You may not know for 2–4 years. Use data (SEMrush), not Claude guesses, for keywords.

Setup: VS Code + Claude Code

  1. Install VS Code (Antigravity no longer supports Claude Code extension — use VS Code instead).
  2. Install the Claude Code extension in VS Code.
  3. Create a project folder (e.g. SEO_brief). Every Claude Code project lives in one folder.
  4. Add CLAUDE.md to the folder root — this is Claude's instruction manual (SSG rules, voice rules, SEO requirements). Copy from the Skool blueprint or clone the GitHub repo.
  5. Clone official repo: git clone https://GitHub.com/jonocatliff/SEO_brief.git

  3. In Claude, click + → upload screenshot from computer.

Index pages: Blog index lists all posts; service index lists all service landing pages.

Static Site Generation (Required)

Google must receive fully rendered HTML instantly when crawling. Use static site generation (SSG) only.

TypeWhat happensSEO
SSGPage pre-built at build time — Google gets HTML immediately✅ Required
Server-side renderingPage built on each request — slower crawl⚠️ Suboptimal
Client-side renderingGoogle may get empty shell — JS builds page in browser❌ Won't rank

CLAUDE.md enforces Next.js output: 'export' (SSG). If Google can't crawl complete HTML, nothing else matters.

Keyword Research — SEMrush (Blog Keywords)

  1. Open SEMrush → SEO → Keyword Magic Tool.
  2. Enter a root keyword (e.g. "plumber") — returns all variations containing that word.
  3. Apply filters:
    • Keyword Difficulty ≤ 30 — you can actually rank
    • Volume ≥ 100/month — worth targeting
    • Intent: Informational — for blog posts (not "buy Nike shoes" commercial intent)
  4. Exclude bad keywords: competitor business names, branded terms, "how to become a [job]" (job-seekers not customers).
  5. Questions tab: great blog ideas (e.g. "how much does a plumber cost?" = buyer intent).
  6. Adjacent topics: top-of-funnel content before they're ready to hire (e.g. "signs you need a new water heater").
  7. Competitor spy: enter competitor domain in SEMrush → see their ranking keywords → steal relevant ones.
  8. Select winners → Apply → add to keyword list (e.g. "plumbing keywords").
  9. Export as CSV → save as keywords.csv in project folder.
  10. Target: a few hundred to a couple thousand keywords.

Don't ask Claude for keywords — it doesn't know difficulty or volume.

Create Blog Post (v2) + Pexels Images

  1. Drag keywords.csv into your Claude Code project.
  2. Run prompt v2 (see prompts.md): pick one primary keyword + 4–5 secondary keywords for a keyword clusters.
  3. After using a primary keyword, add it to used-keywords.md — never reuse.
  4. Pexels API setup:
    • Go to Pexels.com/api → create account → request API key
    • Add to .env: PEXEL_API=your_key_here
  5. Prompt asks Claude to pull images from Pexels via API key in .env.

keyword clusters: One page ranks for many related terms. Root: "how to unclog a drain" + secondaries: "unclog kitchen sink", "slow drain", "home remedy clogged drain", etc. One post can rank for 50–100 keywords.

Fix AI Slop — Voice Files (v3 prep)

First draft will read like generic AI. Fix with reference files in the references/ folder:

  • voice.md — your writing style, sentence rhythm, anti-patterns
  • humour.md — humor rules (first 50 words need a dad joke or self-aware wink; not a brochure)
  • stats.md — real business numbers (jobs completed, years in business, pricing)
  • stories.md — anecdotes from past jobs
  • opinions.md — your takes on the industry

How to populate: Copy your LinkedIn posts, emails, call transcripts, or writing you admire → prompt Claude: "Please update all reference files from these passages."

Why it matters for SEO: Dwell time, bounce rate, and engagement signals. Boring content = people leave = you don't rank.

Rewrite Blog Post — SERP Analysis (v3)

Before rewriting, Claude must research what already ranks:

  1. Search Google for the primary keyword
  2. Find top 3 ranking blog posts (exclude Reddit/UGC)
  3. Analyze and match:
    • Format (listicle, tutorial, guide, comparison)
    • Word count (within 20% of their average)
    • H2 count and image count
    • Every topic all three cover + 1–2 extras they missed
    • Answer the main question in the first paragraph (featured snippet)
    • FAQ from "People Also Ask" (4–8 questions)
  4. Rewrite using humour.md + voice.md + opinions.md + stats.md + stories.md
  5. Images above H2 headings via Pexels

Full prompt in prompts.md v3.

SEO Black Box — How to Win

  • 200+ rumored ranking signals — nobody knows exact weights (not even Google employees).
  • Known signals: dwell time, bounce rate, scroll depth, CTR, content depth, backlinks, technical health.
  • Simple rule: Google wants users to return → serves ads → makes money. Best content wins.
  • #1 lever: Create content people actually want to read (not AI slop).

Service Pages — Zipper Model

Blog posts = topical authority (informational). Service pages = money keywords (commercial intent).

Zipper model: Cross-multiply services × cities:

Services: emergency plumber, drain cleaning, hot water repair
Cities: Vancouver, Toronto, Montreal
→ emergency plumber Vancouver, drain cleaning Toronto, etc.

SEMrush for service keywords:

  1. Keyword Magic Tool → root keyword → sort by Cost Per Click (money keywords; typically $3–$20 CPC)
  2. Look for pattern: service + city or city + service
  3. Export to list → Service-keywords.csv

Build (prompts.md v4): Landing page for city/service — same layout as your tested homepage (Jono got 20% conversion via Google Ads A/B testing).

Limit: Don't create hundreds — similar content across many pages can hurt. Be tasteful (not thousands).

Blog posts raise domain authority → service pages rank higher.

On-Page SEO (v5) — 80+ Signals

Paste the full checklist from on-page-seo.md (or slideshow/onpage-seo-checklist.html) into Claude Code.

Prompt v5: Update blog post with every item from on-page-seo.md — but keep voice and humor. Balance SEO markers with readable content.

Key items verified in demo:

  • 2–3 external links to authoritative sources
  • 3–5 internal links to your other pages (descriptive anchor text)
  • Primary keyword in first 100 words
  • Exactly one H1; proper H2 → H3 hierarchy
  • 4–8 FAQ questions
  • Meta title (blue in Google results) + meta description (gray text)
  • 80+ total signals in full checklist (schema, OG tags, alt text, E-E-A-T, etc.)

Check meta tags: Chrome → three dots → More tools → Developer tools → Elements → search "og:"

Apply to every blog post and service page.

Technical SEO (v6) — Lighthouse 100/100

Three requirements:

  1. sitemap.xml — tells Google every page that exists (submit to Search Console later)
  2. robots.txt — tells Google what to crawl. Allow all public pages; disallow /admin, /dashboard
  3. Google Lighthouse — score 100 on Performance, Accessibility, Best Practices, SEO

Run Lighthouse: Chrome → three dots → More tools → Developer tools → click >> arrows → Lighthouse tab → Analyze page load.

Fix via Claude (prompts.md v6): Copy entire Lighthouse report → paste in Claude → ask to optimize to 100% on all four categories + generate robots.txt and sitemap.xml.

If not 100%, paste remaining issues back and iterate.

Example before fix: Performance 53, SEO 92. After: 100/100/100/96.

Publishing cadence — don't spike volume:

  • Never dump 1,000 posts at once — Google flags volume spikes

Off-Page SEO — What to Avoid & 4 Legit Methods

Jono does not rely on off-page SEO. Cheap backlink services often destroy sites (PBNs = Private Blog Networks = link farms Google detects immediately).

4 legit methods (optional):

  1. Broken backlink swapping: SEMrush → site audit competitor → find their broken external links → offer your page as replacement
  2. Guest posting: Google search "plumbing" "write for us" → write for them, link back to your site
  3. HARO / journalist queries: Answer expert questions → get backlink when published
  4. Paid quality backlinks: Reputable domains only (expensive, pay-to-play on some publications)

Best strategy: Find winning keywords via SEMrush and rank with content — off-page not required.

Deploy: GitHub + Vercel

  1. GitHub: Create new private repo → copy the git commands GitHub shows → in Claude Code: "Please upload my whole project here" + paste repo URL
  2. Vercel: Create free account → connect GitHub → Import project → Preset: Next.js → Deploy (~60 seconds)
  3. Optional: add custom domain via Vercel domain tab (or import from Namecheap)

Localhost = only you can see it. After deploy, site is public.

Post-Launch: Google Business + Search Console

  1. Google Business Profile (Google Business Profile) — free local listing; shows in map pack for "Toronto plumber" type searches
  2. Google Search Console:
    • Add property → URL prefix → paste your Vercel URL (copy full link address)
    • Verify ownership: HTML meta tag method → copy tag → Claude: "Add this meta tag for Search Console verification and redeploy to GitHub + Vercel"
    • Submit sitemap: enter sitemap.xml in Sitemaps section
    • Request indexing: paste a new page URL in top search bar → request indexing (~10 requests/day; can index within 1 day vs weeks)
  3. Google Analytics — track visitor behavior
  4. Landing page testing — A/B test forms/layouts (Google Ads), apply winner to all service pages

Don't dump 1,000 pages on launch day.
